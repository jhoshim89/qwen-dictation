# Qwen Dictation Launch Plan

## Goal

Earn the first 1-10 GitHub stars by clearly explaining the local Mac dictation
workflow to developers and local-AI users. Optimize for real feedback over a
large launch spike.

## Launch order

### Phase 1: GitHub surface

- Rewrite the README first screen around the core promise.
- Use the HUD preview and a concise "How it works" section until demo media is ready.
- Set GitHub repository description, homepage, and topics.
- Keep installation on the default `main` branch.

### Phase 2: X and Threads soft launch

- Post 1-2 short build-in-public updates per day for one week.
- Lead with the workflow, not a generic feature list.
- Reply manually to questions and installation issues.
- Record which phrasing gets profile clicks, repo clicks, and stars.

### Phase 3: Show HN

- Launch after the README explains the workflow and the install flow is stable.
- Use a direct title: `Show HN: Qwen Dictation - local Mac dictation that types into any app`.
- Spend launch day answering comments and fixing obvious docs issues.

### Phase 4: Reddit and follow-up posts

- Post only in relevant communities after reading each community's rules.
- Use a founder/build-note tone, not drive-by promotion.
- Link to GitHub once and answer questions directly.

## Do not automate

- Unsolicited replies to strangers.
- DMs.
- Likes, follows, or stars.
- Reposting the same text across many communities.

## Success metrics

- 1-3 stars from the first soft-launch posts.
- 5-10 stars before deciding whether to do Show HN.
- At least 3 installation or feature questions from real users.
- At least 1 external share by someone other than the maintainer.
