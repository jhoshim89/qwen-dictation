# Channel Playbook

## X

Best use: developer build-in-public posts, workflow explanations, technical tradeoffs.

Posting rules:

- Lead with the workflow.
- Keep the first post under 700 characters.
- Use one GitHub link.
- Do not tag unrelated people.
- Reply manually to every real question in the first 24 hours.

Recommended sequence:

1. Workflow explanation.
2. Why local dictation.
3. Technical note about typing into any app.
4. Qwen/local ASR note.
5. Ask for Mac testers.

## Threads

Best use: lighter version of the X posts.

Posting rules:

- Use simpler language.
- Avoid dense technical detail.
- Use one short paragraph and one link.
- Reuse the same short workflow explanation.

## Show HN

Best use: concentrated feedback from builders.

Posting rules:

- Launch after the README explains the workflow and current MVP status clearly.
- Keep the post factual and direct.
- State that it is a developer MVP.
- Be present for comments during the first 6-8 hours.
- Do not ask friends for upvotes.

Recommended title:

```text
Show HN: Qwen Dictation - local Mac dictation that types into any app
```

## Reddit

Best use: targeted feedback after the README and install flow are stable.

Posting rules:

- Read each community's self-promotion rules first.
- Do not cross-post the same copy everywhere.
- Use a problem/learning framing, not a launch blast.
- Avoid posting in more than 1-2 communities on the same day.

Possible communities to evaluate:

- `r/MacOS`
- `r/LocalLLaMA`
- `r/SideProject`
- `r/programming`
- `r/korea` only if the post is genuinely Korean-language/productivity relevant.
