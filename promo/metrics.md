# Promo Metrics

Track manually at first. The goal is to learn which message produces real
interest, not to optimize vanity numbers.

## Signals to record

- GitHub stars.
- GitHub issues or discussions.
- Installation failures.
- Questions about privacy, model size, latency, or app packaging.
- External shares by people other than the maintainer.
- Which one-line description people repeat back.

## Decision rules

- If people click but do not star, improve README and workflow clarity.
- If people star but do not install, simplify installation.
- If people install but ask the same question, move that answer above the fold.
- If no one engages, change the hook before changing the product.
